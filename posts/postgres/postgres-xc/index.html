pgxc_planner | pgxc_try_planner

  1. 语句可以发送到 Datanode，而且只需要协调器作为代理-在这种情况下，它创建单个节点计划。
  2. 语句可以在协调器上完全计算-因此不涉及查询传送，并且调用 standard _ planner ()来计划语句

https://www.cnblogs.com/yyvv/p/4188980.html

代码的整体质量不错，大部分的改动都有注释，注释可读性也很好，个别注释时效性有问题，但不影响理解代码。所有在pg代码中的改动都用idef做了有效隔离。理论上跟上PG的升级问题不大 postgresql xc修改了一些postgresql的代码，postgresql xl又把他们改了过来，然后又加了好多代码。注意区分#idef和#ifndef Postgresql-xc的原则是能下推到dataNode的就下推到dataNode，实在推不下去的就把所有的数据集中在在聚集节点做。而xl做了MPP。

pgxc_try_planner() {
  pgxc_handle_unsupported_stmts();
  pgxc_handle_exec_direct();
  pgxc_FQS_planner() {
    // 获得语句的执行节点，结果有
    //  1. 无节点
    //  2. 一节点
    //  3. 所有节点
    // 无节点返回的时候，语句判定是不适合ship的，如果有返回节点，则构建执行计划
    exec_nodes = pgxc_is_query_shippable(query, 0) {

      // 遍历语法树，收集sc_context和exec_nodes
      //  sc_context根据节点信息收集shippability
      //  exec_nodes是在pgxc_FQS_get_relation_nodes中计算来的
      pgxc_shippability_walker((Node *)query, &sc_context);
      exec_nodes = sc_context.sc_exec_nodes;

      // 如果语法树中需要一个节点，但是可exec_node不为1，则失败
      if (bms_is_member(SS_NEED_SINGLENODE, shippability)) {
        if (list_length(exec_nodes->nodeids) != 1 && !IsExecNodesReplicated(exec_nodes))
          canShip = false;
      }

      // ShippabilityStat的状态和exec_node一起判断能不能ship，

      if (!canShip && exec_nodes)
        FreeExecNodes(&exec_nodes);
    }

    top_plan = (Plan *)pgxc_FQS_create_remote_plan(query, exec_nodes, false) {
      deparse_query(query, &buf, NIL, true, false);
    }


  }
}
